# 機能追加の取りまとめ

本プロジェクトで実装してきた機能の経緯をまとめたものです。
機能の使い方は [管理者向け機能ガイド](ADMIN-GUIDE.md)、設計の確定事項は [SPEC.md](SPEC.md) を参照してください。

## 初期リリース（2026-06-11）

ランダムマッチング型のビデオ通話Webアプリとして作成。

- ユーザー登録・ログイン（JWT認証）
- ダッシュボード（お知らせ・イベントカレンダー・みんなの掲示板・セッション履歴・オンライン統計）
- ランダムマッチング → 双方同意 → 1対1ビデオ通話（WebRTC）→ 振り返りアンケート
- GitHub連携・Google Cloud Runへの自動デプロイ（pushするだけで反映）

## ブランドリニューアル「対話のおけいこ」（2026-06-11）

企画書に基づき全面リブランド。

- 配色を紺×オレンジ×生成りに変更、文言を「聴く力」コンセプトに統一
- 10分セッションタイマー（時間が来ると自動終了）

## 役割マッチングとセッション呼び名（2026-06-12）

- 「話し手」「聞き手」を選んで待機し、**反対の役割とのみマッチング**
- ダッシュボードに役割別の待機人数を表示
- マッチごとにランダムな**セッション限定の呼び名**を割り振り（本名・ユーザー名は相手に伝えない）
- 通話画面に役割と呼び名を常時表示

## アバター（2026-06-12）

- **デフォルメモード**: MediaPipe顔トラッキング連動の2Dアバター5種（まる・ねこ・うさぎ・くま・ひと）。
  まばたき・口の開閉・笑顔・眉・頭の向き/うなずきに連動。カメラの実映像はネットワークに流れない
- **リアルモード**: 3D VRMアバター5種（ひなた・つむぎ・たける・れん・シード）。すべて再配布可能なライセンスを確認済み
- トラッキング修正: うなずきの上下張り付き（カメラ位置の自動キャリブレーション）、VRM0系モデルの回転反転、構図調整

## ユーザーロールと管理画面（2026-06-12）

- 管理者ロールを導入し、起動時に `administrator` を自動作成
- 管理画面: ユーザー一覧・他ユーザーのセッション履歴閲覧・ユーザー削除・サイト設定（セッション時間・新規登録の許可）・お知らせ管理

## マルチテナント化（2026-06-12）

仕様書（SPEC.md）に基づく4フェーズで実装。

- **フェーズ1: サイト分離** — サイトID単位でユーザー・マッチング・データを完全分離。
  ログイン2系統（メインサイト `/taiwa-lesson`・サブサイト `/login`）、
  4段階ロール（システム管理者＞サイト管理者＞モデレータ＞ユーザー）、
  初期パスワードの初回ログイン時変更強制、サイト管理者の自動生成、
  システム管理画面（サイト作成・削除・各サイトの確認）
- **フェーズ2: 通話と表示の設定** — 表示モード4種（デフォルメ/リアル/静止画/実映像）の利用可否、
  匿名/実名切替、役割なしマッチング、アンケートの有無・設問カスタマイズ
- **フェーズ3: チーム** — 複数所属可のチーム、チームリーダー（招待・削除）、
  チーム限定の掲示板・スケジュール、チーム統計
- **フェーズ4: ルーム** — ルームごとのマッチング（ルーム通話）。モデレータ以上が作成。
  ルーム設定（見える範囲・合言葉・定員・存在期間・ルーム管理者・通話設定の上書き）

## 検討枠5機能（2026-06-12）

- サイト別ブランディング（サイト名・キャッチコピー）
- ユーザーCSV一括登録（パスワード自動生成対応）
- 利用レポート（指標・日別グラフ・チーム別集計）
- 監査ログ（管理操作の記録）
- カレンダー→ルーム連携（予定からワンタップでルーム参加）

## トラスト&セーフティ・通話機能・体験向上（2026-06-12）

- **通報**（理由つき。サイト管理者とチームリーダーのみ閲覧。管理画面に通報キュー）
- **ブロック**（この相手と二度とマッチしない）
- **警告文**（サイト管理者・チームリーダーが発令。対象のログイン時に画面中央ポップアップ）
- **パスワードリセット**（サイト管理者・モデレータが新しい初期パスワードを発行）
- 通話機能: **ミュート・映像オフ・画面共有・セッション内チャット**（利用可否はサイト設定）。
  チャットログは終了画面で保存を促してから削除
- **役割交代つきセッション**（10分×2回。時間終了時に双方同意で役割を交代してもう1回）
- マッチ成立の通知音
- **話題カード**（ルーム設定／話し手が設定／ロビーはなし・ランダム・任意。独自アセット対応）
- **デバイステスト**（カメラ・マイクの選択、プレビュー、音量メーター、アバターのお試し）
- レポートCSVエクスポート、アンケート複数設問化、メール連携（SMTP設定時）、
  「また話したい」相互一致の再マッチ優先

## PWA化（2026-06-12）

- スマホのホーム画面に追加してアプリのように起動可能（manifest・アイコン・Service Worker）
- VRMモデルとアイコンのみキャッシュし、コード類は常に最新を取得

- **ルームをカード表示でダッシュボード上部に配置**。カードに名前・話題・参加人数/定員・
  セッション時間・役割を表示
- **ルームカード押下で詳細モーダルを表示**（話題・参加状況・マッチング方式・表示モード・

- **すべてのモーダルが背景クリックで閉じる**ように統一

## チーム画面とマイページ（2026-06-13）

- **チーム画面を新設** — ダッシュボードの「マイチーム」やマイページから「開く」で遷移。
  管理画面に入れない**チームリーダーの管理導線**になる
  - チーム情報（名前・**説明文**(新設)・統計）／メンバー一覧／チーム掲示板／今月のチーム予定／
    リーダー向けの通報一覧
  - リーダーはチーム設定の変更・メンバー招待・削除・警告ができる
- **チームリーダーの複数人化** — リーダーが共同リーダーを任命/解除できる
  （自分自身の権限変更は不可。サイト管理者は全操作可）
- **チームからの脱退**（セルフ削除）。最後のリーダーは他のリーダーを任命するまで脱退不可
- **マイページを新設** — 右上のユーザー名をクリックで表示。プロフィール
  （ユーザー名・権限・サイト・登録日・セッション回数）、メールアドレスの変更、
  パスワード変更、セッション履歴、所属チーム（チーム画面への導線）

## テーマ切替（2026-06-13）

- **5種類の配色テーマ** — スタンダード（紺×コーラル）・ダーク・さくら・わかば・ラベンダー。
  マイページの「テーマ」から色見本つきボタンで選択し、即時反映
- 選択は `localStorage`（`vm_theme`）に端末ごとに保存。CSS読込前に適用してちらつきを防止
- 実装はCSSカスタムプロパティの一括上書き（`<html data-theme="...">`）。
  PWAのステータスバー色（`theme-color`）もテーマに追従

## システム管理・ヘッダー・表示名の再設計（2026-06-13）

- **ヘッダー刷新**: 左上のサイト名クリックでホームへ戻る。右上はユーザーアイコン（👤表示名▾）の
  ドロップダウンメニュー（設定=マイページ／ログアウト）。**システム管理ボタンは金色の特別な
  見た目**にして、全サイト横断の特権操作であることを明示
- **システム管理画面の再構成**:
  - サイト一覧のCSVエクスポート。サイト名クリックで**サイト詳細ページ**へ遷移
  - サイト詳細は**「ユーザー一覧」「設定確認」の2タブ**。ユーザー一覧の各行に「編集」
    （表示名・ロール・有効/無効をモーダルで変更）。設定確認タブの「編集」でサイト設定と
    同じ全項目を編集可能（`PUT /api/sysadmin/sites/{id}/settings`）
  - 「サイト名を変更」でサイトの表示名を変更（サイトID=slugは紐付け用のため変更不可）
- **チーム管理のモーダル化**: 管理画面でチーム名クリック→詳細モーダル（内部ID・説明・統計・
  メンバー）→「編集」でチーム名・説明・リーダー任命/解除・メンバー追加/削除。
  編集ボタンは権限（リーダーまたはサイト管理者）がある場合のみ表示
- **表示名（display_name）の導入**: ログインID（username）と表示用の名前を分離。
  マイページ・管理画面・システム管理画面から変更可能。掲示板・イベント・チーム・通報・
  通話相手名・ルーム作成者などの表示はすべて表示名を使用（内部の紐付けは従来どおり整数IDのFK）
- 管理画面ユーザー一覧も「詳細→編集」パターンに統一（ロール変更・有効/無効は編集モーダルへ集約）

## 本日の予定・イベント時刻・出欠制マッチング（朝会）（2026-06-13）

- **イベントに開始時刻**（任意）を追加。カレンダー一覧・本日の予定に時刻つきで表示
- **本日の予定バナー**: ダッシュボードの「おかえりなさい」とルームの間に、当日の予定だけを
  表示（サイト全体 + 自分の所属チーム）。`/api/events/today?date=` で取得
- **参加可否（RSVP）**: 各イベントに「参加／不参加」をユーザーごとに登録（`EventAttendance`）。
  本日バナーから切り替え、参加予定人数も表示
- **イベント→ルーム入室**: ルーム連携イベントは本日バナー・カレンダーから「入室する」で
  ルーム入室モーダルを開ける
- **出欠制マッチング（朝会）**: ルームに「出欠制」フラグを追加。ONにすると、その日に
  そのルームへ紐づくイベントで「参加」を選んだ人だけが待機列に入れる
  （`Room.attendance_required`）。チーム（見える範囲）と本日のイベントを組み合わせると、
  **参加可能なチームメンバーだけをランダムにマッチング**できる
  - WebSocketの `join_queue` にクライアントのローカル日付を渡して当日判定。出欠未登録・
    不参加・本日イベントなしは入室をブロックし、理由を返す
- デモデータに「デモ_朝会ルーム」（営業チーム限定・出欠制）+ 本日の朝会イベント +
  メンバーのRSVPの例を追加

## 繰り返しイベント（Googleカレンダー風）（2026-06-13）

- イベント作成時に**繰り返し**を選べるように: 毎日／平日（月〜金）／毎週／毎月
- **繰り返し終了日**を指定（空欄なら約3か月先まで。1シリーズ最大366回）
- 各回を実体化して同じ `series_id` でまとめる（各回ごとにRSVP・出欠制ルームの当日判定が
  そのまま機能する。朝会を「平日・毎朝」で一度に仕込める）
- カレンダー一覧・本日の予定に **🔁** と繰り返しラベル（毎日／平日／毎週○／毎月○日）を表示
- **イベント削除**を新設（`DELETE /api/events/{id}?scope=one|series`）。繰り返しは
  「この回だけ」「繰り返し全体」を選択。削除できるのは作成者・サイト管理者・
  （チーム予定なら）そのチームのリーダーのみで、権限があるイベントにだけ削除ボタンを表示

## イベント詳細モーダル・編集（2026-06-13）

- カレンダーの日付に**マウスオーバーすると概要**（その日の予定の時刻・タイトル）が出る
- 予定を**クリックすると詳細モーダル**を表示（日時・繰り返し・公開範囲・連携ルーム・
  参加予定・作成者）。1日に複数あるときは日別一覧モーダルから選択
- 詳細の「**編集**」で日時・タイトル・公開範囲・ルーム連携を変更（`PUT /api/events/{id}`）。
  繰り返しは「この回だけ／繰り返し全体（日付以外）」を選択
- 編集ボタンは権限のある人（作成者・サイト管理者・チームリーダー）にのみ表示。
  詳細から入室・削除もできる
- カレンダー一覧・本日の予定のタイトルはクリックで詳細を開ける
