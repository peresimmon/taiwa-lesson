# 対話のおけいこ マルチテナント化 仕様書

2026-06-12 ヒアリング確定版。

## 概念

- **サイトID**でユーザー・マッチング・設定・掲示板等をすべて分離する
- **メインサイト**: サイトID `taiwa-lesson`（変更の可能性あり）。「対話のおけいこ」として一般向けに展開
- **サブサイト**: 各企業向け「社内コミュニケーションツール」として展開

## ログイン

| ページ | URL | 入力項目 |
|---|---|---|
| メインサイト | `/taiwa-lesson` | ユーザーID・パスワード |
| サブサイト | `/login` | サイトID・ユーザーID・パスワード |

- URL文字列は変更の可能性あり
- ユーザーIDは**サイト内で一意**（別サイトに同名ユーザーが存在できる）
- 自己登録: メインサイトのみ可（デモ運用のため。本番では廃止予定）。サブサイトは管理者によるアカウント作成のみ

## ユーザーロール

| ロール | 権限 |
|---|---|
| システム管理者 | 全サイト・全ユーザーの管理。最高権限。メインサイトの「システム管理画面」を利用可能 |
| サイト管理者 | サイト内の全機能・全ユーザーの管理。ユーザーの作成・削除はサイト管理者のみ。サイト作成時に `{サイトID}_admin` / `password@{サイトID}` で自動生成 |
| モデレータ | ルーム作成・ルーム設定（基本は自分が作ったルームのみ管理）。サイト全体をざっくり回す人員。※個別ユーザーの割り当ては行わない（管理対象の概念はチーム/チームリーダー側に持たせる） |
| ユーザー | 一般ユーザー。掲示板への書き込み、スケジュール作成が可能 |

- **初期パスワードのアカウントは初回ログイン時にパスワード変更を強制**（administrator含む）
- モデレータの権限: 履歴閲覧・パスワードリセット等。削除・追加は不可（サイト管理者のみ）

## 管理画面

- **サイト管理画面**（各サイト内・サイト管理者用）: サイト設定、ユーザー管理など
- **システム管理画面**（メインサイトのみ・システム管理者用）: 全サイトの作成・削除、各サイトの設定・ユーザーの確認

## チーム

- ユーザーをチーム単位で管理。**複数チームへの所属可**
- 所属の設定は**サイト管理者のみ**
- チームに**チームリーダー**を設定可能。チームリーダーは自チームへのユーザー招待・チームからの削除が可能
- チーム限定のスケジュール、チーム限定ルームを作成できる
- 掲示板・カレンダー・統計はサイト単位とチーム単位それぞれで確認可能

## ルーム

- 通常マッチングとは別に、**ルームごとにマッチング**を行う
- モデレータ以上が作成可能。基本は自分が作ったルームのみ管理
- **ルーム設定**（すべてルームごとに変更可能。サイト設定より優先）
  - 見える範囲（全員/チーム限定）
  - 入る際の合言葉
  - 定員
  - 存在期間
  - ルーム管理者（設定されたユーザーはルーム設定を変更可能）
  - 通話設定の上書き（セッション時間・表示モード・**役割マッチング有無も上書き可**）

## サイト設定

### 通話設定
- 通話セッション時間（分）
- 通話画面の表示モード: **実映像 / リアルモードアバター / デフォルメモードアバター / 静止画** の4種類それぞれの使用可否

### サイト設定
- 役割（聞き手/話し手）マッチング or 役割なしマッチング
  - 役割なしの場合、アバターの色分けはランダムで良い
- 匿名化コミュニケーションの有無（呼び名のランダム割当 ⇔ ユーザーID表示）
- ルーム作成機能のオンオフ
- アンケートの有無・設問内容の変更（サブサイト向け）
- 新規登録の許可（メインサイトのみ。サブサイトは常に不可）
