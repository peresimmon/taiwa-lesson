# 管理者向け機能ガイド

このサービスを運営する方のための、機能の説明書です。プログラミングの知識は不要です。

---

## 1. このサービスについて

**対話のおけいこ**は、匿名の相手と1対1のビデオ通話で「聴く力」を練習するサービスです。
1つのシステムの中に複数の「サイト」を作ることができ、一般向けの**メインサイト**と、
企業ごとの**サブサイト**（社内コミュニケーションツール）を分けて運営できます。
サイト同士は完全に分かれており、ユーザーや通話相手が混ざることはありません。

### 用語

| 用語 | 意味 |
|---|---|
| メインサイト | 「対話のおけいこ」として一般公開するサイト。`/taiwa-lesson` からログイン |
| サブサイト | 企業向けのサイト。`/login` からサイトID・ユーザー名・パスワードでログイン |
| ロビー通話 | ダッシュボードの「相手を探す」ボタンから行う通話 |
| ルーム通話 | 「ルーム」に入った人同士で行う通話 |
| セッション | 1回の対話（通話）のこと |
| アバター | 通話で顔の代わりに表示されるキャラクター。実際の顔は相手に送信されません |

---

## 2. ロール（権限）

| ロール | できること |
|---|---|
| **システム管理者** | すべて。サイトの作成・削除、全サイトの確認（システム管理画面） |
| **サイト管理者** | 自分のサイト内のすべて。ユーザーの作成・削除、サイト設定、お知らせ、チーム管理、通報対応、警告、パスワードリセット |
| **モデレータ** | ルームの作成・自分が作ったルームの管理、パスワードリセット |
| **チームリーダー** | 自チームへのユーザー招待・削除、自チームメンバーへの警告、自チームに関する通報の閲覧 |
| **ユーザー** | 通話、掲示板への投稿、予定の作成 |

- サイトを作ると、サイト管理者のアカウント（`サイトID_admin`）が自動で作られます
- モデレータへの昇格は、サイト管理者が管理画面のユーザー一覧から行います
- チームリーダーは、サイト管理者がチーム管理から任命します

### 初期パスワードについて（重要）

自動作成されたアカウント、管理者が作成したアカウント、パスワードリセット後のアカウントは、
**初回ログイン時に必ずパスワードの変更**を求められます。変更が済むまで他の機能は使えません。

※例外として、メインサイトの `administrator` だけは**デモ版に限り**変更を求められません
（本番運用では変更強制を復活させます）。

## 3. 利用者から見た流れ

1. **ログイン** — メインサイトは自分で登録も可能（サブサイトは管理者が作ったアカウントのみ）
2. **ダッシュボード** — 上部に**ルームのカード一覧**（カードをタップすると詳細モーダルが開き、
   「マッチング開始」または「話し手／聞き手として参加」ボタンで待機へ。合言葉もモーダル内で入力）、
   その下にお知らせ・カレンダー・掲示板・マイチーム・自分の履歴が並びます。
   カレンダーの予定は「イベントを作成」ボタンか日付のクリックで開くモーダルから登録します。
   **右上のユーザーアイコン（👤）を押すとメニュー**が開き、「設定（マイページ）」「ログアウト」が
   選べます（後述）。**左上のサイト名を押すといつでもホームに戻れます**
3. **相手を探す** — 「話し手」「聞き手」を選んで待機（サイト設定で役割なしにもできます）。
   待機中はカメラ・マイク・アバターのテストができます
4. **マッチング** — 相手が見つかると通知音が鳴り、確認画面へ。お互いの呼び名
   （匿名サイトではランダムな呼び名、実名サイトではユーザー名）と対話のルールが表示されます。
   ここで**表示モード**（アバターの種類など）を選び、話し手は**今日の話題**を設定できます
5. **通話** — 双方が同意すると開始。残り時間・呼び名・役割・話題カードが常に表示されます。
   ミュート／映像オフ／画面共有／チャットが使えます（サイト設定で制限可能）
6. **時間終了** — 役割ありの通話では「交代してもう1回」を選べます（双方が同意した場合のみ）
7. **アンケート** — 振り返りを記録。「また話したい」を付けると、お互いに付けた相手と
   次回優先的にマッチします。チャットを使った場合はログの保存を促し、画面を閉じると削除されます
8. **困ったとき** — 通話中・終了後に**通報**でき、終了後に**ブロック**（二度とマッチしない）もできます

### 表示モード（通話画面に映るもの）

| モード | 内容 |
|---|---|
| デフォルメモードアバター | かわいい2Dキャラクター5種。表情や頭の動きが連動 |
| リアルモードアバター | 3Dキャラクター5種。よりリアルな見た目で表情が連動 |
| 静止画 | 呼び名のカードを表示し、音声のみで対話 |
| 実映像 | カメラの映像をそのまま表示（社内利用など向け。選択時に警告を表示） |

アバター・静止画では、カメラ映像は表情の読み取りだけに使われ、**相手に実際の顔は送信されません**。

## 4. チーム

ユーザーをチームでまとめられます（複数チームへの所属可）。

- チームの作成・削除は**サイト管理者**が管理画面から行います
- ダッシュボードの「マイチーム」やマイページからチームを**「開く」と専用のチーム画面**が表示されます。
  メンバー一覧・チーム掲示板・今月のチーム予定・チーム統計が見られます
- **チームリーダー**はチーム画面から次の操作ができます（管理画面に入る必要はありません）:
  - チーム設定の変更（チーム名・説明文）
  - メンバーの招待・削除・警告
  - **共同リーダーの任命・解除**（リーダーは複数人設定できます。自分自身の権限は変更不可）
  - このチームへの通報の確認
- メンバーは自分でチームから**脱退**できます（最後のリーダーは、他のリーダーを任命するまで脱退できません）
- 掲示板とカレンダーは、パネル右上の切替で「サイト全体」と「チーム限定」を行き来できます。
  チーム限定の投稿・予定はチーム外の人には見えません

## 4-2. マイページ

右上のユーザーアイコン（👤）→「設定（マイページ）」で開きます。

- **プロフィール** — ユーザー名（ログインID）・表示名・権限・サイト・登録日・セッション回数
- **表示名の変更** — 掲示板・チーム・通話相手への表示に使われる名前。ログインIDは変わりません（空欄でユーザー名と同じ）
- **メールアドレスの変更** — パスワードリセットの案内などに使われます（空欄で削除）
- **パスワード変更**
- **テーマ** — 画面の配色を5種類から選択（端末ごとに保存）
- **セッション履歴** — 自分の振り返りの一覧
- **所属チーム** — そのままチーム画面を開けます

## 5. ルーム

通常のロビー通話とは別の「部屋」を作って、入った人同士だけでマッチングできます。

- 作成できるのは**モデレータ以上**（サイト設定でルーム機能ごと無効にもできます）
- ルームごとに設定できる項目: 見える範囲（全員／チーム限定）・入る際の合言葉・定員・
  存在期間・話題カード・**通話設定の上書き**（セッション時間・役割の有無・表示モード）・
  **出欠制マッチング**（後述）
- 「ルーム管理者」に設定されたユーザーも設定を変更できます
- カレンダーの予定にルームを紐づけると、予定からワンタップで参加できます

### 出欠制マッチング（朝会などに）

チームの朝会のように「その日参加できる人だけで通話したい」場合に使います。

1. **ルームにチームを紐づけ**（見える範囲＝チーム限定）、「**出欠制にする**」をオンにします
2. その日の**イベントを作成し、このルームを紐づけ**ます（毎朝1件作るイメージ。時刻も設定可）
3. メンバーは**本日の予定バナー**から「参加／不参加」を選びます
4. 「参加」を選んだメンバーだけがルームに入室でき、その人どうしでランダムにマッチングされます
   （不参加・未回答の人、当日のイベントが無い場合は入室できません）

---

## 5-2. 本日の予定とイベント

- イベントには**日付と時刻（任意）**を設定できます
- ダッシュボード上部の「**本日の予定**」に、その日の予定（サイト全体＋自分の所属チーム）が
  時刻つきで表示されます
- 各予定で「**参加／不参加**」を選べます（参加予定の人数も表示）
- ルームを紐づけた予定は「**入室する**」ボタンでそのままルーム入室モーダルを開けます

### 詳細と編集

- カレンダーの日付に**マウスを乗せる**と、その日の予定の概要（時刻・タイトル）が出ます
- 予定を**クリックすると詳細モーダル**が開きます（日時・繰り返し・公開範囲・連携ルーム・
  参加予定・作成者）。1日に複数あるときは一覧から選びます
- 詳細の「**編集**」（作成者・サイト管理者・チームリーダーに表示）で、日時・タイトル・
  公開範囲・ルーム連携を変更できます。繰り返しイベントは「この回だけ／繰り返し全体」を選べます
- 本日の予定やカレンダー一覧では、タイトルをクリックしても詳細が開きます

### 繰り返し（Googleカレンダー風）

- イベント作成時に**繰り返し**を選べます: 毎日／平日（月〜金）／毎週／毎月
- **繰り返し終了日**を指定できます（空欄なら約3か月先まで）
- 朝会のように「平日・毎朝」の予定を一度にまとめて作れます。各回ごとに参加可否を選べます
- 繰り返しイベントには 🔁 が付きます
- **削除**ボタン（作成者・サイト管理者・チームリーダーに表示）で、繰り返しは
  「この回だけ」「繰り返し全体」を選んで削除できます

## 6. 管理画面（サイト管理者／チームリーダー用）

ログイン後、ヘッダーの「🗂 管理画面」から開きます。上部のタブで
**「ユーザー」「サイト設定」「レポート・ログ」「チーム・お知らせ」**の4ページを切り替えます。
なお、各種モーダル（履歴・警告など）は背景の灰色部分をクリックしても閉じられます。

### チームリーダーの場合（管理対象のみのスコープ表示）

サイト管理者でない**チームリーダー**も管理画面を開けますが、表示は
**自分が率いるチームとそのメンバー（管理対象）に絞り込まれます**。

- **ユーザー**: 自分が率いるチームのメンバーのみ（閲覧のみ。作成・編集・削除などの操作はありません）
- **サイト設定**: 表示されません（サイト管理者のみ）
- **レポート・ログ**: 管理対象ユーザーのみの集計。通報も自分のチームのメンバー宛のみ。監査ログは表示されません
- **チーム・お知らせ**: 自分が率いるチームのみ表示。チームの詳細・編集（メンバー・リーダー・名前）は可能。
  チーム作成・削除はサイト管理者のみ。お知らせは自分のチーム限定のみ投稿できます（後述）

以下はサイト管理者の操作説明です（チームリーダーは上記の範囲に限られます）。

### ユーザー（一覧は画面の横幅いっぱいに表示）
- 列見出しのクリックで**ソート**できます（もう一度押すと昇順⇄降順。▲▼で表示）
- ユーザーの作成（メールアドレスを入れると案内メールを送信※）・削除
- **CSV一括登録** — 1行に「ユーザー名,初期パスワード,メール」。パスワードを省略すると
  自動生成され、その場で一度だけ表示されます
- **編集** — モーダルで**表示名・ロール（一般／モデレータ／サイト管理者）・有効/無効**をまとめて変更
  （システム管理者と自分自身のロールは変更不可。管理者は無効化不可。無効化中は行がグレー表示）
- そのほかの操作: **履歴**（セッション履歴をモーダルで表示）/ **PWリセット**（新しい初期パスワードを発行）/
  **警告**（次回ログイン時にポップアップで表示される警告文を送る）
- **CSVエクスポート** — ユーザー一覧（権限・状態・メール・セッション数つき）をダウンロード

### 通報
利用者からの通報が届きます。「対応済みにする」で処理し、必要なら「警告」ボタンで
そのまま対象者に警告文を発令できます。

### 利用レポート
登録者数・セッション数・直近の利用・平均評価・日別グラフ・チーム別の集計。
「CSVエクスポート」でセッションデータをExcelで開ける形式でダウンロードできます。

### 監査ログ
管理者・モデレータが行った操作（設定変更、ユーザー作成・削除、警告など）の記録です。
「誰が・何を・いつ」行ったかを後から確認できます。

### チーム管理
チームの作成・削除。**チーム名を押すと詳細モーダル**（内部ID・説明・統計・メンバー）が開き、
「編集」からチーム名・説明の変更、リーダーの任命・解除、メンバーの追加・除名ができます
（編集ボタンは権限がある場合のみ表示）。

### お知らせ管理
ダッシュボードに表示されるお知らせの投稿・削除。投稿時に**表示範囲**を選べます。

- **サイト全体**: 全員のダッシュボードに表示（サイト管理者のみ投稿可）
- **〇〇チーム限定**: そのチームのメンバーにだけ表示（そのチームのリーダー、またはサイト管理者が投稿可）

お知らせ一覧・ダッシュボードでは、チーム限定のものにチーム名タグが付きます。
チームリーダーは自分が率いるチーム限定のお知らせのみ投稿・削除できます。

**本文で使える変数**: 本文に `{roomN}`（Nはルーム番号）と書くと、表示時にそのルームへの
リンクになり、押すとルーム詳細が開きます。投稿フォームの「本文」横の **?** を押すと、
使える変数と現在のルーム番号一覧が確認できます。

**閲覧側の動作**: ダッシュボードのお知らせは省略表示で、タイトルを押すと詳細（全文）が開きます。
詳細では前後の矢印で他のお知らせへ移動でき、開くと既読になります（未読は「新着」バッジ表示）。
**投稿権限のある人**は、詳細で「送信対象人数」「既読人数」「未読人数」を確認できます。
既読数・未読数を押すと、それぞれ**既読・未読のユーザー一覧**が開きます。

### サイト設定

設定は「サイト全体の設定」「マッチングの設定」「通話の設定」「通話前に使用する設定」
「アンケートの設定」の5つのセクションに分かれています。保存ボタンは共通で、1回の保存で全項目が反映されます。

| 設定 | 内容 |
|---|---|
| サイト名 / キャッチコピー | ヘッダーやダッシュボードの表示が変わります（サブサイトを自社名にできます） |
| セッション時間 | 1回の通話の長さ（分） |
| 新規ユーザー登録 | 自己登録の受付（メインサイト用。サブサイトは常に管理者作成のみ） |
| 役割マッチング | 「話し手×聞き手」で組むか、役割なしで組むか |
| 匿名コミュニケーション | ON=ランダムな呼び名 / OFF=ユーザー名をそのまま表示 |
| 通話機能の利用可否 | ミュート・映像オフ・画面共有・チャットの個別ON/OFF |
| 役割交代つきセッション | 時間終了時の「交代してもう1回」のON/OFF |
| 再マッチ優先 | 「また話したい」同士を優先してマッチ |
| ルーム作成機能 | ルーム機能全体のON/OFF |
| 表示モード | 4種それぞれの利用可否（最低1つは必要） |
| ロビー通話の話題カード | なし／ランダム表示／任意設定。ランダムは内蔵の話題に加え、独自の話題リストも登録可能 |
| アンケート | 実施の有無、設問（1問または複数問） |

※メール送信にはサーバー側のメール設定（SMTP）が必要です。未設定の間は画面表示のみになります。

## 7. システム管理画面（システム管理者用）

メインサイトに `administrator` でログインすると、ヘッダーに**金色の「🛡 システム管理」ボタン**が
表示されます（全サイトを横断する特権操作のため、通常のボタンと見た目を変えています）。
上部のタブで**「サイト一覧」「サイト作成」「サイトの詳細」**を切り替えます。

- **サイト一覧** — 全サイトとユーザー数の確認と**CSVエクスポート**。**サイト名を押すと
  「サイトの詳細」へ移動**します。サイトの削除もここから
  （サイトと所属ユーザー・データをすべて削除。メインサイトは削除不可）
- **サイトの詳細** — **「ユーザー一覧」「設定確認」の2タブ**
  - ユーザー一覧: 各行の「編集」で表示名・ロール・有効/無効をモーダルで変更
  - 設定確認: サイト設定の全項目を一覧表示。「編集」を押すとその場で
    サイト設定と同じ項目をすべて変更できます
  - 「サイト名を変更」でサイトの表示名を変更（サイトID=slugは内部の紐付け用のため変更不可）
- **サイト作成** — サイトID（半角英数とハイフン）とサイト名を入れて作成。
  サイト管理者アカウントと初期パスワードが**その場で一度だけ**表示されるので、
  必ず控えて先方の管理者に渡してください

### デモデータ（開発者ページに移動）

デモデータの生成・削除は **開発者ページ（🛠 開発者）** に移動しました（7-2 を参照）。

## 7-2. 開発者モード（DBブラウザ）

**システム管理者がメインサイトにログインしたとき**だけ、ヘッダーに緑色の **🛠 開発者** ボタンが
表示されます（システム管理と同じく、特別な操作だとわかる見た目にしています）。

- データベースの各テーブルの中身を**参照**できる開発・調査向けのツールです
- テーブル一覧（行数・列数つき)から選ぶと、ビューア（モーダル）が開きます
- ビューアでは **カラム・演算子（=・含む・≧ など）・値** の条件を追加して絞り込み、
  列見出しのクリックで並び替え、表示件数を指定できます
- **安全のための制限**:
  - 参照（SELECT）のみ。データの追加・変更・削除はできません
  - SQLを直接書くことはできず、画面で組んだ条件だけが安全に実行されます
  - 1回の表示は最大1000行まで。`password_hash` など機微な値は `***` で伏せられます

### デモデータ（開発者ページ内・デモ版限定）

開発者ページの「デモデータ」パネルで、動作確認用のデータをワンクリックで生成・削除できます。
**あらゆる権限・状態を一通り試せるよう、幅広いパターンのデータが作られます。**

- **デモユーザー12名**（デモ_さくら など。パスワードは全員 `demo1234`）。権限・状態を網羅:
  - **サイト管理者** … デモ_たろう（管理画面の全機能を試せます）
  - **モデレーター兼チームリーダー** … デモ_さくら（営業チームのリーダー。リーダー向け管理画面・チーム限定お知らせを試せます）
  - **モデレーター** … もう1名
  - **無効化されたアカウント** … 1名（ログイン拒否の確認用）
  - 表示名（display_name）付き・メールアドレス付きなど、プロフィールの違いも混在
- **チーム4種**: 営業／開発（共同リーダー）／人事／**administratorがリーダーの「デモ_管理者チーム」**
- **ルーム8種**（すべてルーム番号付き）: 雑談・合言葉（`aikotoba`）・定員2名・役割なし・期間限定・
  管理者追加・**出欠制の「デモ_朝会ルーム」**・**出欠制の「デモ_管理者ルーム」**
- **イベント**: 本日のイベント（ルーム連携）・過去/未来の単発・毎日/毎週/毎月の繰り返し・
  チーム限定イベントなど。administrator のダッシュボードからそのまま試せます
- **お知らせ**: 全体向け・`{room番号}` でルームにリンクするお知らせ・営業チーム限定のお知らせ。
  一部は既読済みにしてあり、既読人数・新着バッジの確認ができます
- **通話履歴・アンケート・通報（未対応/対応済み）・ブロック・警告** も一通り
- **デモ用サブサイト「demo-corp（デモ株式会社）」**（管理者 `demo-corp_admin` /
  `password@demo-corp` でそのままログイン可能。実名表示ON・実映像ONの別設定サイト）

- **生成・削除の仕組み**: デモデータは DB に直接書き込むのではなく、**実際のユーザーや管理者が
  画面から操作するのと同じ API ルートを通って**作成・削除されます。そのため本番同様の
  バリデーション・監査ログ・付随処理がすべて働き、削除時も孤児データが残りません
- **削除** — 生成したデータをサブサイトごとまとめて削除します。実際のユーザーには影響しません
- この機能は本番運用では撤去されます

## 8. よくある運用手順

**新しい会社（サブサイト）を始める**
1. システム管理画面でサイトを作成 → 表示された管理者アカウントを先方に渡す
2. 先方のサイト管理者が `/login` からログイン（初回にパスワード変更）
3. サイト設定を会社向けに調整（例: 実名表示ON・実映像ON・役割なし・自社名に変更）
4. CSV一括登録で社員のアカウントを作成し、ID・初期パスワードを配布

**「パスワードを忘れた」と言われた**
管理画面のユーザー一覧で対象者の「PWリセット」→ 表示された新しい初期パスワードを本人に伝える
（本人は次のログインで自分のパスワードに変更します）

**迷惑行為の通報が来た**
管理画面の「通報」で内容を確認 → 「警告」で警告文を発令（次回ログイン時にポップアップ表示）→
「対応済みにする」。悪質な場合はユーザー一覧から削除。
なお、通報した本人は「ブロック」で自衛できます（その2人は二度とマッチしません）

**スマホでアプリのように使いたい**
サイトをスマホで開き、Android(Chrome)は「アプリをインストール」、
iPhone(Safari)は共有ボタン→「ホーム画面に追加」。

## 9. 注意事項

- **デモ環境はデータが消えることがあります**（サーバーの再デプロイでリセット）。
  本格運用時の対応事項は開発側で管理しています（docs/TODO.md）
- `administrator` の初期パスワードは初回ログイン時に必ず変更してください
- 企業ネットワークなど一部の環境では、通話がつながりにくい場合があります
  （本番運用時にTURNサーバーという仕組みで解消予定）
