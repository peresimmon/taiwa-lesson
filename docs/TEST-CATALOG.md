# テスト内容一覧

自動結合テスト（`tests/test_flow.py`）で検証している項目の一覧です。
**全366項目**を機能単位のセクションに分けて毎回チェックしています。

## 実行方法

```powershell
# テスト用DBでサーバーを起動（ポート8000）
$env:DATABASE_URL = "sqlite:///./test_videomatch.db"
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000

# 別ターミナルで実行
python tests/test_flow.py
```

- 実サーバーに対してHTTP/WebSocketで実際にリクエストを投げる**結合テスト**です（モックではありません）
- 各項目は `check(名前, 条件)` で判定し、`結果: NNN passed, M failed` を出力します
- セクションは下記の実行順で走ります（`main()` の呼び出し順）。`print("[番号]")` の番号は歴史的経緯でとびとびですが、本書では実行順に 1〜25 で整理しています

## 補足

- 上記の項目数は「ソース上の `check()` 呼び出し」を整理したものです。一部はループ（例: ユーザー登録×2、初回パスワード変更フロー）で実行時に複数回判定されるため、**実行時の合計は366項目**になります。
- このテストは実DBに対して動作します。デモデータの生成・削除も**実APIルート経由**で行われ、削除後に `call_pairs` / `reports` / `blocks` / `warnings` / `announcement_reads` / `event_attendance` / `events` / `surveys` に孤児が残らないことを別途スモークテストで確認しています。
